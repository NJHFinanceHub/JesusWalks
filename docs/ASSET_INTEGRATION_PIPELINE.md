# Unreal Asset Integration Pipeline

## Purpose
Define a production pipeline for importing and validating art/audio assets so the project can scale without content debt.

## Scope
- Environment art (static meshes, materials, textures)
- Character content (skeletal meshes, animations)
- VFX/audio assets
- Validation and release gating

## Directory and naming standards

### Core content roots
- `unreal/TheNazareneAAA/Content/Art/Environment/<Region>/...`
- `unreal/TheNazareneAAA/Content/Art/Characters/<Faction>/<Character>/...`
- `unreal/TheNazareneAAA/Content/Art/VFX/...`
- `unreal/TheNazareneAAA/Content/Art/Audio/...`
- `unreal/TheNazareneAAA/Content/UI/...`

### Required prefixes
- `SM_` static mesh
- `SK_` skeletal mesh
- `M_` material
- `MI_` material instance
- `T_` texture
- `A_` animation sequence
- `NS_` Niagara system
- `S_` sound wave
- `BP_` blueprint

## Import checklist

### Environment meshes
1. Use centimeter scale, Z-up, and freeze transforms before export.
2. Author at least 3 LODs for hero meshes and 2 LODs for props.
3. Use simple collision (UCX or autogenerated) for gameplay-blocking assets.
4. Keep pivot placement intentional (snap/grid friendly for modular kits).

### Character assets
1. Standardize skeleton hierarchy and naming before retarget.
2. Validate root motion flags and animation compression settings.
3. Keep mesh/material slot count minimal and consistent across variants.

### Materials and textures
1. Use channel packing for masks where possible.
2. Set compression and sRGB correctly by map type.
3. Ensure master-material driven instances for most surface variants.

### Audio/VFX
1. Route audio through project-defined classes/mixes.
2. Normalize loudness and avoid per-asset gain hacks.
3. Validate Niagara bounds and scalability tiers.

## Validation automation
- Use `unreal/TheNazareneAAA/Tools/validate_asset_pipeline.py` in CI or local checks.
- The validator currently enforces:
  - Naming prefix by class type.
  - Minimum LOD recommendations for static meshes.
  - Presence of simple collision for static meshes.

## Command examples

### Headless validation
```bat
UnrealEditor-Cmd.exe TheNazareneAAA.uproject -run=pythonscript -script=Tools/validate_asset_pipeline.py -unattended -nop4
```

### Manual run from Python console
```python
exec(open("Tools/validate_asset_pipeline.py").read())
```

## Release gate policy
- Block release candidates when validator reports critical failures.
- Allow warnings temporarily only with explicit waiver in release notes.
- Track waivers by asset path and expiry date.
